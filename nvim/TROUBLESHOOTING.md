# Neovim トラブルシューティング

問題が発生した時に使えるデバッグコマンド集。

## 基本的な確認

| コマンド | 用途 |
|----------|------|
| `:messages` | エラーや通知の履歴 |
| `:checkhealth` | プラグインや環境の診断 |
| `:version` | Neovimのバージョン・ビルド情報 |

## キーマッピング関連

| コマンド | 用途 |
|----------|------|
| `:map <key>` | 特定キーのマッピング確認（例: `:map <leader>`） |
| `:verbose map <key>` | どのファイルで定義されたかも表示 |
| `:nmap` / `:imap` / `:tmap` | モード別のマッピング一覧 |

## 状態確認

| コマンド | 用途 |
|----------|------|
| `:mode` | 現在のモード |
| `:set <option>?` | オプションの値確認（例: `:set timeoutlen?`） |
| `:verbose set <option>?` | どこで設定されたかも表示 |
| `:registers` | レジスタの内容 |
| `:marks` | マーク一覧 |

## プラグイン関連

| コマンド | 用途 |
|----------|------|
| `:Lazy` | lazy.nvimのUI（プラグイン状態確認） |
| `:LspInfo` | LSPの接続状態 |
| `:Mason` | Masonでインストールしたツール一覧 |

## デバッグ用

| コマンド | 用途 |
|----------|------|
| `:lua print(vim.inspect(...))` | Lua変数の内容を表示 |
| `:echo &option` | オプション値を表示 |
| `:scriptnames` | 読み込まれたスクリプト一覧 |

## よくある問題

### leaderキー（Space）が効かない

1. モードを確認: ノーマルモードでないと動作しない
2. `:mode` で現在のモードを確認
3. `<Esc>` を押してノーマルモードに戻る
4. ターミナルモードの場合は `<C-x>` で抜ける

### キーマッピングが期待通りに動かない

1. `:verbose map <key>` でどこで定義されたか確認
2. 複数のプラグインが同じキーをマッピングしている可能性
3. `:checkhealth which-key` でwhich-keyの状態確認

### プラグインが読み込まれない

1. `:Lazy` でプラグインの状態確認
2. `:checkhealth` で依存関係の問題を確認
3. `:messages` でエラーを確認
